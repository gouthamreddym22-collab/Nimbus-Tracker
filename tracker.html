Nimbus — Progress

Working title — product name TBD

Nimbus is a continuous personal AI companion — accessible across phone, desktop, and (over time) every form factor you live with — that develops a persistent relationship with each user over years. One Nimbus per user. Knows you, stays with you, quietly works alongside your life.

Architecturally anchored on the Universal Conversion Substrate (every internal substrate is rotatable without breaking user state), the Nimbus Standard Library (federated capability schemas that compound across users), trustless vault + universal Action Audit Log (privacy by structure, not promise), the Private Inference Substrate (Premium-tier sealed enclave for private-data inference), and a unified Notification/Reminder/Confirmation substrate every vertical consumes.

This tracker measures honest progress toward two product rungs. V1 — single-client demo: phone + PC on a local PC server with Claude OAuth; essential 5-verb surface; 3 starter verticals; demoable to one prospect at a time; no partner-compute dependency. Alpha — closed alpha: 50–100 invited users across consumer archetypes; full 8-vertical kernel; 3-tier matrix; partner-compute data centre.

◆ How progress is scored. Each submodule carries a phase tag (Forever / Pre-Alpha-gate / V1 / V1.5 / Alpha / Beta+ / Long-horizon) and four build dimensions — Vision / Implementation / Code / Test — each worth 25% of the submodule's effective percent. Rung percentages are cumulative: V1 % averages every sub-item whose phase is Forever or V1; Alpha % averages every sub-item whose phase is Forever, Pre-Alpha-gate, V1, V1.5, or Alpha. Alpha therefore includes V1 — Alpha cannot regress below V1 unless additional Alpha-scope work weighs the average down. Until dedicated test passes run, modules are not yet measurable; the rung percentages carry the untested-work discount.
